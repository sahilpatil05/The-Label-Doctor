6.1 IMAGE SCANNING & INGREDIENT EXTRACTION
    Workflow:
      1. User uploads food product image OR captures via camera
      2. EasyOCR extracts text from image
      3. Extracted text parsed for ingredients
      4. Ingredients cleaned and normalized
      5. Results returned to frontend
    
    Supported Formats: JPG, PNG, WebP
    Processing:
      - Image validation
      - OCR processing
      - Text extraction
      - Ingredient parsing

6.2 ALLERGEN DETECTION
    Features:
      - Real-time allergen identification
      - Cross-reference with allergen database
      - Severity classification:
        * Critical
        * Warning
        * Minor
        * Safe
      - Multi-allergen detection
      - User-specific allergen filtering
    
    Detection Logic:
      - Pattern matching against allergen list
      - Fuzzy matching for synonym detection
      - Severity scoring
      - Risk assessment

6.3 HEALTH SCORE CALCULATION
    Methodology:
      - Ingredient analysis
      - Allergen count impact
      - Ingredient quality assessment
      - Nutritional consideration
    
    Scoring:
      - 0-30: Poor (High allergen/risky)
      - 31-60: Fair (Some concerns)
      - 61-80: Good (Mostly safe)
      - 81-100: Excellent (Very safe, healthy)

6.4 PRODUCT RECOMMENDATIONS

================================================================================
                        7. API ENDPOINTS
================================================================================

7.1 CORE ENDPOINTS

    GET /
      Purpose: Serve main HTML page
      Response: HTML page with embedded CSS/JS

    POST /api/scan
      Purpose: Upload and scan food label image
      Request:
        - Form data: image file
        - Optional: user_id for personalization
      Response:
        {
          "success": true,
          "ingredients": [list of ingredients],
          "detected_allergens": [allergen list],
          "warnings": [warning list],
          "health_score": 82.5,
          "is_safe": true,
          "alternatives": [safe products]
        }

    POST /api/analyze
      Purpose: Analyze extracted ingredients
      Request:
        {
          "ingredients": [ingredient list],
          "user_allergens": [allergen list]
        }
      Response:
        {
          "safe_ingredients": [list],
          "risky_ingredients": [list],
          "health_score": 78.0,
          "warnings": [list]
        }

    GET /api/allergens
      Purpose: Get complete allergen database
      Response:
        {
          "allergens": {
            "category": [allergen list]
          }
        }

    GET /api/alternatives/<allergen>
      Purpose: Get safe alternatives for allergen
      Response:
        {
          "allergen": "milk",
          "alternatives": [product list]
        }

7.2 USER MANAGEMENT ENDPOINTS

    POST /api/user/create
      Purpose: Create new user profile
      Request:
        {
          "name": "John Doe",
          "email": "john@example.com",
          "allergens": ["milk", "gluten"],
          "dietary_preferences": {"vegan": false}
        }
      Response:
        {
          "success": true,
          "user_id": "uuid",
          "message": "User created successfully"
        }

    GET /api/user/<user_id>
      Purpose: Retrieve user profile
      Response:
        {
          "id": "uuid",
          "name": "John Doe",
          "email": "john@example.com",
          "allergens": ["milk", "gluten"],
          "dietary_preferences": {...},
          "created_at": "2026-03-18T..."
        }

    PUT /api/user/<user_id>
      Purpose: Update user profile
      Request:
        {
          "allergens": ["milk", "gluten", "peanut"]
        }
      Response:
        {
          "success": true,
          "user_id": "uuid",
          "message": "User updated"
        }

7.3 CAMERA ENDPOINTS

    GET /api/camera/stream
      Purpose: Stream live video feed (Motion JPEG)
      Response: Continuous video stream
      Content-Type: multipart/x-mixed-replace

    POST /api/camera/capture
      Purpose: Capture and process single frame
      Response:
        {
          "frame": "base64_encoded_image",
          "timestamp": "2026-03-18T20:15:00"
        }

7.4 INSIGHTS & ANALYTICS ENDPOINTS

    GET /api/insights/<user_id>
      Purpose: Get user statistics and scan history
      Response:
        {
          "total_scans": 5,
          "average_health_score": 78.5,
          "most_common_allergens": [allergen list],
          "scan_history": [scan list]
        }

    GET /api/scan/<scan_id>/insights
      Purpose: Get detailed insights for specific scan
      Response:
        {
          "total_ingredients": 15,
          "safe_ingredients": 12,
          "risky_ingredients": 3,
          "safety_percentage": 80.0,
          "ingredient_breakdown": {...}
        }

7.5 PRODUCT ENDPOINTS

    GET /api/products
      Purpose: Browse product database
      Query params:
        - user_id: Filter allergen-safe products
        - category: Filter by category
        - sort_by: "health_score" or "rating"
      Response: List of products

    GET /api/products/<product_id>
      Purpose: Get detailed product information
      Response:
        {
          "id": "uuid",
          "name": "Product Name",
          "ingredients": [...],
          "allergen_free_for": [allergen list],
          "health_rating": 8.5,
          "category": "Snacks"
        }

    GET /api/recommendations/<user_id>
      Purpose: Get personalized product recommendations
      Response: List of recommended products
